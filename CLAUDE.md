# CLAUDE.md

This is a text-only reference repository. There is no code to build or test.

## What This Is

An opinionated git workflow framework meant to be consumed by humans and AI agents. It defines branching strategy, commit conventions, PR policy, release flow, artifact management, environment promotion, and channel-based rollout.

## Reading Order

1. **README.md** — Philosophy, meta-pattern, and document index
2. **GLOSSARY.md** — Ubiquitous language. Adopt this vocabulary exactly.
3. **WORKFLOW.md** — Day-to-day development rules: branches, PRs, commits, issues
4. **RELEASES.md** — Delivery operations: tags, versions, artifacts, channels, promotion, hotfixes
5. **ADAPTING.md** — Operational tiers (solo/personal, solo/consultancy, small team) and product type variations

## When Referencing This Framework

- Determine which tier applies to the target project (see ADAPTING.md)
- Follow the tier-appropriate level of ceremony
- Use the glossary terms consistently
